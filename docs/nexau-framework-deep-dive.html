💧 五、数据流详解

📨 消息流转过程

  • 用户输入字符串 "明天下午 3 点开会"
  • 转换为 Message 对象 Message(role=USER, content=[TextBlock(text="...")])
  • 添加到 HistoryList history.add_message()
  • 转换为 OpenAI 格式 [{"role": "user", "content": "..."}]
  • 发送给 LLM API openai.ChatCompletion.create()
  • 接收 LLM 响应 ChatCompletionMessage(tool_calls=[...])
  • 解析为 ToolCall 对象 ToolCall(name="add_event", args={...})

🔧 工具执行流程

  • 解析 ToolCall 提取 tool_name 和 parameters
  • 查找工具 tool = tool_registry[tool_name]
  • 执行 BeforeTool 钩子 middleware.before_tool()
  • 参数验证 jsonschema.validate(params, tool.input_schema)
  • 调用绑定函数 tool.implementation(**parameters)
  • 执行 AfterTool 钩子 middleware.after_tool()
  • 返回结果 {"status": "success", "event_id": "..."}

📚 六、关键概念解释

🎯 Executor（执行器）
Executor 是框架的"引擎"，负责整个执行循环：调用 LLM → 解析响应 → 执行工具 → 检查结果 → 重复直到完成。它是 Agent 类的"大脑"，处理所有核心逻辑。
# agent.py 中创建 Executor self.executor = Executor( agent_name=self.config.name, tool_registry=self.tools, llm_config=self.config.llm_config, ... )
🔌 Middleware（中间件）
中间件是在 LLM 调用前后、工具执行前后插入的"钩子"。可以用来做日志、缓存、修改请求/响应等。类似 Web 框架的 Middleware。
# hooks.py 定义钩子 class BeforeModelHook: async def __call__(self, input: BeforeModelHookInput): # 可以修改 messages、tools 等 return input
📦 Tool Registry（工具注册表）
工具注册表是一个字典，key 是工具名，value 是 Tool 对象。Executor 通过它查找要执行的工具。类似"电话簿"。
# executor.py 初始化 self.tool_registry = { "add_event": Tool(...), "query_event": Tool(...), ... }
📝 HistoryList（历史列表）
HistoryList 保存所有对话历史（用户消息、AI 回复、工具调用结果）。每次调用 LLM 时都会带上历史，让 AI 有"记忆"。
# history_list.py class HistoryList: def add_user_message(self, text: str): self.messages.append(Message(role=USER, content=text)) def to_openai_messages(self) -> list[dict]: # 转换为 OpenAI API 格式 ...
🔀 Tool Call Mode（工具调用模式）
NexAU 支持多种工具调用格式：OpenAI 的 function calling、Anthropic 的 XML 格式、自定义 XML 格式。tool_call_mode 参数决定用哪种。
# config.yaml tool_call_mode: "openai" # 或 "xml"、"anthropic" # executor.py 中转换 if self.tool_call_mode == "openai": tools = [t.to_openai_tool() for t in self.tools]
🎛️ GlobalStorage（全局存储）
GlobalStorage 是在整个 Agent 执行过程中共享的数据存储。工具可以读取/修改它，实现跨工具的状态共享。
# agent_context.py class GlobalStorage: def __init__(self): self._data = {} def get(self, key: str) -> Any: return self._data.get(key) def set(self, key: str, value: Any): self._data[key] = value

💡 总结：NexAU 框架核心思想

Agent 是"外壳" → Executor 是"引擎" → ToolExecutor 是"执行器"

🎯
职责分离
Agent 负责配置，Executor 负责执行，Tool 负责具体功能
🔌
可扩展性
通过中间件、钩子、自定义工具轻松扩展功能
📦
模块化设计
每个组件独立，可以单独替换/升级
🔄
循环执行
LLM → 解析 → 执行 → 检查 → 重复，直到完成任务
