🤖 个人日程助手 - 智能体架构

基于 NexAU 框架的日程管理智能体系统架构

💬
用户交互层
User Interface Layer
main.py Python
CLI 入口，处理用户输入并启动对话循环
system-prompt.md Markdown
定义智能体角色、能力和行为规范
config.yaml YAML
智能体配置（模型、工具、上下文限制）
🧠
智能体核心层
Agent Core Layer
NexAU Framework Python
Agent 框架核心：上下文管理、工具调度、对话状态
LLM Router Python
模型路由：Qwen / DeepSeek / 硅基流动
Tool Dispatcher Python
解析 LLM 意图，调用对应工具
🛠️
工具层
Tool Layer
add_event YAML + Python
创建新日程，参数：title, start_time, end_time
modify_event YAML + Python
修改现有日程，参数：event_id + 更新字段
query_event YAML + Python
查询日程，参数：date_range, keyword
delete_event YAML + Python
删除日程，参数：event_id
⚙️
核心业务逻辑层
Business Logic Layer
EventManager Python
日程 CRUD 操作、ID 生成、数据验证
ScheduleParser Python
自然语言时间解析、标准化（明天 3 点 → 2026-03-10 15:00）
DocParser Python
从 Markdown/聊天记录提取日程信息
💾
数据持久化层
Data Persistence Layer
events.json JSON
日程数据存储，按 event_id 索引
.env Env
环境变量：LLM_API_KEY, LLM_BASE_URL 等
logs/ Log
运行日志、调试信息
🌐
外部服务层
External Services
LLM Provider API
Qwen / DeepSeek / 硅基流动（OpenAI 兼容 API）
Langfuse Optional
可观测性平台（可选）：追踪、监控、分析
📊 请求处理流程
1
用户输入
"明天下午 3 点开会"
2
NexAU 解析
上下文 + 意图识别
3
LLM 推理
工具选择 + 参数提取
4
工具调用
add_event → EventManager
5
数据持久化
写入 events.json
6
返回结果
"已创建日程"
